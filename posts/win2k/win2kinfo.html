Windows 2000 Kernel Extensions + Installation

On this part of the page, I will include several kernel extension installations. What I'm refering to a kernel extension is a modification of the kernel with the intention of being able to run newer programs, or newer versions of programs, that date from Windows XP on Windows 2000. The reason it is possible is because the kernels between Windows 2000 and Windows XP aren't necessarily different, so, in a way, it's quite an easier step to modify the Windows 2000 kernel than to modify the Windows XP kernel in order for most Vista programs to run on XP, as XP and Vista kernels are majorly different.

blackwingcat's Extended Kernel

A popular kernel extension is blackwingcat's (or BWC's) Extended Kernel, which is based on optimizing kernel32.dll more, including several other files. If you're gonna want to install that, then make sure to have the available updates for Windows 2000 installed. Win32's and qashqai's ISO shouldn't be a problem, as both have updates already slipstreamed.

The most popular version used is v3.0e, although the latest one, as of now, v3.1a2, is only available as a .wlu file. Fortunately, BWC has prepared already a wlupdater, that allows the installation from .wlu files.

Download v3.0e (i430vx's Mirror)

In order to download the latest version of BWC's Extended Kernel, that is only available on win2k.org, hosted by BWC himself. After downloading the .exe or .wlu, simply run the file. If it's been downloaded as .wlu though, you're gonna have to use the wlupdater to run files with the .wlu extension. Most probably, the installer will ask you if you want to install Root Certificate updates and a customized version of Windows Installer, click "Yes" on both message boxes. If everything's met, then click on the "Execute" Button to install the extended kernel.

Next time when you startup with BWC's Extended Kernel installed for the first time, you'll encounter an error shown on a message box. Ignore it, it will only show up on that one time. So, afterwards, you shouldn't see the error pop up. If everything's been fine as it is now, congratulations, you installed a blackwingcat's Extended Kernel which will allow you to run many XP related programs on Windows 2000.

I know there are other folders as you can see in that third image. I'll explain some that can't be understood immediately.

- Compatibility Forcer, tricks the installers into thinking they're running on Windows XP, though it doesn't work with many installers.
- fcwin088, a more advanced program that actually forces the installers to run by doing the same trick as the Compatiblity Forcer does, but it's slightly more complicated to use.

Now, if you don't necessarily want to use fcwin088 because every single time it requires manual changing of the compatibility, like me, you can always use an OS Spoofer. It writes the User Environment Variable, so that a majority of installers and some programs think that they run on XP, without having to individually modify each file with fcwin088. For example, Blender (as the program itself, but probably installer included) needs for the OS Spoofer to write the UEV so it thinks that it's running on XP, in order to start up Blender in the first place, but that is a particularly rare example, as most programs after installation aren't required for the OS Spoofer to be set to XP. Keep in mind that installing the programs initially does require the OS Spoofer. For more information about OS Spoofer, check here.
